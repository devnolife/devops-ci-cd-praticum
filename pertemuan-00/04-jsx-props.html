Konsep 04

Component, Props & JSX

Membangun UI dari potongan-potongan kecil yang bisa dipakai ulang

🎯 Analogi

Bayangkan Lego!

🧱

Component = Block Lego

Setiap block Lego punya bentuk dan warna sendiri. Kamu bisa ambil satu block, lalu pakai berkali-kali di banyak tempat. Itulah Component — potongan UI yang bisa dipakai ulang.

Props = label yang tertempel di block. Label itu bilang: "warna: merah, ukuran: besar". Dengan label berbeda, block yang sama bisa terlihat berbeda!

JSX = cara kamu menulis instruksi perakitan block di dalam kode. Seperti buku panduan Lego yang bilang block mana dipasang di mana.

💡 Bayangkan membangun rumah Lego

Setiap bata (component) bisa dipakai ulang. Desain bata yang sama hadir dalam warna dan ukuran berbeda (props). Buku instruksi yang memberitahu cara merangkai bata itulah JSX. Kamu menulis instruksi sekali, lalu React yang merangkainya menjadi tampilan di layar.

📖 Penjelasan

Apa Itu Component, JSX, dan Props?

🧩

Component

Fungsi JavaScript yang mengembalikan tampilan UI

📝

JSX

HTML yang bisa dicampur dengan ekspresi JavaScript

📦

Props

Data yang dikirim dari parent ke child component

🧩 Component

  • Component adalah fungsi JavaScript yang mengembalikan UI dalam bentuk JSX.
  • Nama component HARUS diawali huruf BESAR (PascalCase): Button, ProfileCard, Navbar.
  • Bisa dipakai berulang kali — inilah konsep reusable.
  • Biasanya disimpan di folder components/.
  • Satu component = satu file. Contoh: components/Button.jsx.
Struktur folder
// Contoh struktur folder project Next.js
app/
├── page.js
├── layout.js
└── components/
    ├── Navbar.jsx
    ├── KartuMahasiswa.jsx
    └── Footer.jsx

📝 JSX — HTML + JavaScript

JSX terlihat seperti HTML, tapi sebenarnya itu JavaScript. Berikut perbedaannya:

  • Gunakan className bukan class
  • Kurung kurawal {} untuk ekspresi JavaScript: {nama}, {2 + 3}, {kondisi ? "ya" : "tidak"}
  • Self-closing tags WAJIB: <img />, <br />, <input />
  • Harus punya 1 parent element — wrap dengan <div> atau <>...</> (Fragment)
  • Inline style pakai double curly braces: style={{ color: 'red' }}

Perbandingan cepat HTML vs JSX:

HTML Biasa

<div class="kartu">
  <img src="foto.jpg">
  <p style="color: red">
    Halo Dunia
  </p>
  <br>
</div>

JSX (React)

<div className="kartu">
  <img src="foto.jpg" />
  <p style={{ color: 'red' }}>
    {salam}
  </p>
  <br />
</div>

📦 Props — Data dari Parent ke Child

  • Props adalah data yang dikirim dari parent ke child component.
  • Mirip seperti parameter fungsi — kamu kirim data, component menerima dan menampilkannya.
  • Props bersifat read-only — child TIDAK bisa mengubah props yang diterima.

Dua cara menerima props:

Tanpa Destructuring

function Card(props) {
  return <h2>{props.title}</h2>;
}

Dengan Destructuring ✨

function Card({ title, value }) {
  return <h2>{title}</h2>;
}
💡 Default Value

Kamu bisa memberi nilai bawaan untuk props: function Button({ color = "blue" }). Kalau parent tidak mengirim color, nilainya otomatis "blue".

⚙️ Mekanisme

Bagaimana Cara Kerjanya?

🌳 Pohon Component (Component Tree)

Perhatikan bagaimana data (props) mengalir dari atas ke bawah, dari parent ke child. Titik-titik violet yang bergerak melambangkan aliran props.

🔤 Ekspresi di Dalam JSX

Di dalam JSX, kurung kurawal {} adalah "jendela ke JavaScript". Apa pun yang ada di dalam {} akan dieksekusi sebagai JavaScript.

Berbagai ekspresi JSX
// 1. Menampilkan variabel
const nama = "Budi";
<h1>Halo, {nama}!</h1>          // → Halo, Budi!

// 2. Ekspresi matematika
<p>Hasil: {2 + 3}</p>          // → Hasil: 5

// 3. Kondisional (ternary)
const sudahLogin = true;
<p>{sudahLogin ? "Selamat datang!" : "Silakan login"}</p>

// 4. Render daftar dengan .map()
const buah = ["Apel", "Mangga", "Jeruk"];
<ul>
  {buah.map(item => <li key={item}>{item}</li>)}
</ul>

// 5. Akses properti object
const user = { nama: "Siti", umur: 21 };
<p>{user.nama} berumur {user.umur} tahun</p>

// 6. Template literal
<p>{`Halo, ${nama}! Selamat datang.`}</p>

⬇️ Aliran Data Satu Arah (One-Way Data Flow)

Dalam React, data mengalir dalam satu arah: dari parent ke child. Ini disebut one-way data flow atau top-down data flow.

  • Parent mengirim data ke child melalui props
  • Child TIDAK bisa mengirim data balik ke parent melalui props
  • Props bersifat read-only — child tidak boleh mengubah props
  • Kalau child perlu mengubah data, gunakan state (akan dipelajari nanti)
⚠️ Ingat!

Props hanya mengalir ke bawah, tidak pernah ke atas. Jika kamu butuh data mengalir ke atas (dari child ke parent), kamu akan belajar tentang callback function dan state di pertemuan berikutnya.

🎁 Children — Props Spesial

Ada satu props spesial bernama children. Apa pun yang kamu tulis di antara tag pembuka dan penutup component akan menjadi children.

Children prop
// Apa pun di antara <Wrapper> dan </Wrapper> = children
<Wrapper>
  <h1>Ini adalah children!</h1>
  <p>Konten apa saja bisa masuk di sini.</p>
</Wrapper>

// Di dalam component Wrapper:
function Wrapper({ children }) {
  return (
    <div className="wrapper">
      {children}  {/* konten akan muncul di sini */}
    </div>
  );
}
💡 Kapan pakai children?

Gunakan children saat kamu membuat component "wadah" — seperti Card, Modal, Dialog, atau Layout. Component ini tidak peduli apa isinya, ia hanya menyediakan bungkusan (style, border, dll).

💻 Contoh Kode

Belajar dari Contoh

Berikut 6 contoh kode mulai dari yang paling sederhana sampai yang lebih kompleks. Perhatikan komentar di setiap baris untuk memahami apa yang terjadi.

1️⃣ Component Paling Sederhana

components/Sapaan.jsx
// Komponen paling sederhana — hanya return JSX
// Tidak menerima props apa pun

export default function Sapaan() {
  return <h1>Halo, Dunia!</h1>;
}

// Cara pakai di file lain:
// import Sapaan from './components/Sapaan';
// <Sapaan />  → akan tampil: Halo, Dunia!

2️⃣ Component dengan Props

components/Sapaan.jsx — Menerima props
// Menerima props sebagai parameter
// Akses data lewat props.namaData

export default function Sapaan(props) {
  return <h1>Halo, {props.nama}!</h1>;
}

// Cara pakai — kirim data lewat atribut:
<Sapaan nama="Budi" />    // → Halo, Budi!
<Sapaan nama="Siti" />    // → Halo, Siti!

3️⃣ Props dengan Destructuring

components/KartuMahasiswa.jsx
// Destructuring props langsung di parameter fungsi
// Lebih rapi daripada menulis props.nama, props.nim, dst.

export default function KartuMahasiswa({ nama, nim, jurusan }) {
  return (
    <div className="kartu">
      <h2>{nama}</h2>
      <p>NIM: {nim}</p>
      <p>Jurusan: {jurusan}</p>
    </div>
  );
}

// Cara pakai:
<KartuMahasiswa
  nama="Budi Santoso"
  nim="105841100001"
  jurusan="Teknik Informatika"
/>

4️⃣ Render Daftar dengan .map()

app/page.js — Render daftar mahasiswa
// Array berisi data mahasiswa
const mahasiswa = [
  { nama: "Budi", nim: "105841100001" },
  { nama: "Siti", nim: "105841100002" },
  { nama: "Andi", nim: "105841100003" },
];

export default function Home() {
  return (
    <div>
      <h1>Daftar Mahasiswa</h1>

      {/* .map() mengubah array jadi daftar component */}
      {mahasiswa.map((mhs) => (
        <KartuMahasiswa
          key={mhs.nim}      {/* key WAJIB unik! */}
          nama={mhs.nama}
          nim={mhs.nim}
        />
      ))}
    </div>
  );
}
💡 Kenapa perlu key?

React menggunakan key untuk melacak setiap item di daftar. Tanpa key, React bingung item mana yang berubah, ditambah, atau dihapus. Gunakan ID unik (seperti NIM) sebagai key.

5️⃣ Mengirim Object sebagai Props

Kirim object sebagai satu prop
// Daripada mengirim banyak props satu-satu...
// Kamu bisa mengirim satu object utuh!

const profil = {
  nama: "Budi",
  umur: 20,
  hobi: ["Coding", "Gaming"],
};

// Kirim sebagai satu prop bernama "data"
<KartuProfil data={profil} />

// Di dalam KartuProfil:
function KartuProfil({ data }) {
  return (
    <div>
      <h2>{data.nama}</h2>
      <p>Umur: {data.umur}</p>
      <p>Hobi: {data.hobi.join(", ")}</p>
    </div>
  );
}

6️⃣ Children Props (Membungkus Konten)

components/Kotak.jsx — Menggunakan children
// Component "wadah" yang membungkus konten apa saja

export default function Kotak({ warna, children }) {
  return (
    <div style={{ border: `2px solid ${warna}`, padding: 16 }}>
      {children}
    </div>
  );
}

// Cara pakai — konten di antara tag jadi children:
<Kotak warna="blue">
  <h2>Judul di dalam kotak</h2>
  <p>Konten apa saja bisa masuk ke sini!</p>
</Kotak>

// Kotak yang sama, isi berbeda:
<Kotak warna="red">
  <p>Kotak merah dengan konten berbeda</p>
</Kotak>
⚖️ Perbandingan

HTML Biasa vs React/JSX

Kalau kamu sudah familiar dengan HTML, berikut perbedaan utama yang perlu kamu ingat saat menulis JSX.

HTML Biasa React / JSX Keterangan
class="btn" className="btn" class adalah reserved word di JS
for="email" htmlFor="email" for juga reserved word di JS
<br> <br /> Self-closing wajib di JSX
onclick="fn()" onClick={fn} camelCase + curly braces
style="color: red" style={{ color: 'red' }} Object, bukan string
Teks statis {ekspresi dinamis} JSX bisa menyisipkan JS
💡 Tips mengingat

Aturan utamanya: di JSX, semua atribut memakai camelCase (onClick, className, htmlFor, tabIndex). Dan semua tag harus ditutup, termasuk <img /> dan <input />.

⚠️ Perhatian

Kesalahan Umum Pemula

Hampir semua pemula pernah melakukan kesalahan-kesalahan ini. Yuk pelajari biar kamu tidak mengulanginya!

1. Nama Component Huruf Kecil

React membedakan component (huruf besar) dan tag HTML biasa (huruf kecil).

✗ function card() { return <div>...</div>; } — Dianggap tag HTML <card>, bukan component!
✓ function Card() { return <div>...</div>; } — Huruf besar = dikenali sebagai component

2. Lupa Kurung Kurawal { }

Tanpa kurung kurawal, JSX menampilkan teks biasa, bukan nilai variabel.

✗ <h1>Halo, props.nama</h1> — Tampil literal: "Halo, props.nama"
✓ <h1>Halo, {props.nama}</h1> — Tampil nilai: "Halo, Budi"

3. Mengubah Props Secara Langsung

Props bersifat read-only. Mencoba mengubahnya akan menyebabkan error.

✗ props.nama = "Budi"; — AKAN crash! Props tidak bisa diubah.
✓ Props bersifat read-only. Gunakan state jika perlu mengubah data.

4. Lupa key di .map()

Setiap element yang di-render dengan .map() membutuhkan prop key unik.

✗ data.map(item => <Card nama={item.nama} />) — React warning: "Each child should have a unique key prop"
✓ data.map(item => <Card key={item.id} nama={item.nama} />)
🚫 Jangan abaikan warning di console!

Warning tentang key mungkin terlihat sepele, tapi bisa menyebabkan bug yang sulit dilacak — seperti list yang tidak update dengan benar saat data berubah.

Ayo Praktik!

Coba kerjakan latihan berikut untuk memperkuat pemahamanmu tentang Component, Props, dan JSX.

1

Buat Component KartuProfil

Buat component KartuProfil yang menerima 3 props: nama (string), peran (string), dan foto (URL gambar). Tampilkan foto sebagai <img>, nama sebagai heading, dan peran sebagai paragraf. Gunakan destructuring untuk menerima props.

2

Render Daftar Profil dengan .map()

Buat array berisi 3 object profil (masing-masing punya id, nama, peran, dan foto). Lalu render semuanya menggunakan .map() dan component KartuProfil dari latihan 1. Jangan lupa prop key!

3

🔥 Challenge: Component Tombol Dinamis

Buat component Tombol yang menerima props: teks (string), warna (string untuk background), dan ukuran ("kecil", "sedang", atau "besar"). Render 5 tombol berbeda — variasikan warna, teks, dan ukurannya. Gunakan inline style style={{ ... }} untuk menerapkan warna dan ukuran.

  • ✅ Kurung kurawal {} untuk ekspresi JavaScript di dalam JSX

  • ✅ Props bersifat read-only — tidak boleh diubah oleh child
  • ✅ Selalu gunakan key yang unik saat render list dengan .map()
