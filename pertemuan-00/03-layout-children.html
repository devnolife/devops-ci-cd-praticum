Layout & Children

Bingkai yang membungkus setiap halaman

Analogi lain: pikirkan sebuah museum. Ketika kamu berjalan dari satu ruangan ke ruangan lain, bangunannya — dinding, lantai, pencahayaan — tetap sama. Yang berubah hanyalah karya seni di setiap ruangan.

💡 Layout = bangunan museum. Pages = karya seni di setiap ruangan. Kamu tidak perlu membangun ulang gedungnya setiap berpindah ruangan!

📖 Apa Itu Layout?

Dalam Next.js App Router, layout adalah file khusus yang membungkus (wrap) semua halaman di dalam folder yang sama. Kamu menulisnya sekali, dan efeknya berlaku untuk semua halaman di bawahnya.

  • layout.js di folder app/ disebut Root Layout — ini WAJIB ada di setiap project Next.js.
  • Root layout harus me-return tag <html> dan <body>.
  • {children} adalah prop khusus — isinya adalah halaman yang sedang aktif saat ini (placeholder).
  • CSS yang di-import di layout berlaku secara global untuk semua halaman yang dibungkusnya.
  • Metadata (title, description) bisa didefinisikan langsung di layout menggunakan export const metadata.

⚠️ Setiap project Next.js WAJIB punya minimal satu app/layout.js. Tanpa file ini, aplikasi kamu tidak akan berjalan!

⚙️ Bagaimana Cara Kerjanya?

Langkah Demi Langkah

Begini cara Next.js memproses layout secara internal:

  1. Next.js membaca file layout.js di folder app/.
  2. Layout ini otomatis membungkus SEMUA halaman (page.js) di folder tersebut dan subfolder-subfoldernya.
  3. Ketika user berpindah halaman (navigasi), hanya bagian {children} yang di-render ulang.
  4. Navbar dan footer TIDAK di-render ulang — ini membuat navigasi terasa sangat cepat! ⚡
  5. Next.js secara otomatis memasukkan konten page.js sebagai {children} ke dalam layout.

Secara konseptual, inilah yang dilakukan Next.js di belakang layar:

Ilustrasi internal Next.js
// Kamu menulis layout.js dan page.js terpisah.
// Next.js secara OTOMATIS menggabungkannya seperti ini:

<RootLayout>              <!-- dari app/layout.js -->
  <nav>Navbar</nav>         <!-- bagian tetap -->

  <HomePage />             <!-- ini = {children} -->
                             <!-- dari app/page.js -->

  <footer>Footer</footer> <!-- bagian tetap -->
</RootLayout>

Diagram: Nested Layout (Layout Bersarang)

Layout bisa bersarang (nested). Misalnya, kamu punya layout global untuk seluruh website, dan layout khusus untuk halaman dashboard yang punya sidebar.

Nested Layout — Penjelasan

  • app/layout.js = Root Layout → Navbar global & Footer global. Berlaku untuk seluruh website.
  • app/dashboard/layout.js = Dashboard Layout → Sidebar khusus untuk section dashboard. Hanya muncul di halaman /dashboard/*.
  • Cara mereka bersarang: Root membungkus Dashboard, yang membungkus Page.
  • Sidebar dashboard TIDAK muncul di halaman lain (misal /about atau /contact), karena ia hanya ada di dalam dashboard layout.
Struktur folder nested layout
app/
├── layout.js          ← Root layout (navbar + footer)
├── page.js            ← Halaman utama (/)
├── about/
│   └── page.js        ← Halaman tentang (/about)
└── dashboard/
    ├── layout.js      ← Dashboard layout (sidebar)
    ├── page.js        ← Dashboard home (/dashboard)
    ├── nilai/
    │   └── page.js    ← Halaman nilai (/dashboard/nilai)
    └── jadwal/
        └── page.js    ← Halaman jadwal (/dashboard/jadwal)

Metadata Export

Next.js menggunakan metadata export untuk mengatur <title> dan <meta> tags di head dokumen HTML. Kamu cukup export sebuah objek bernama metadata:

app/layout.js — Metadata
// Metadata yang di-export di layout berlaku untuk SEMUA halaman
export const metadata = {
  title: 'Website Kampus',
  description: 'Portal informasi mahasiswa',
};

💡 Metadata di halaman (page.js) akan otomatis meng-override metadata di layout. Jadi kamu bisa punya title berbeda untuk setiap halaman!

💻 Contoh Kode

1. Root Layout (Wajib Ada!)

Ini adalah file layout paling penting. Setiap project Next.js WAJIB punya file ini di folder app/.

app/layout.js
// app/layout.js — Layout utama (WAJIB ada di setiap project!)
import './globals.css'         // CSS global di-import di sini

// Metadata untuk seluruh website
export const metadata = {
  title: 'Website Kampus',
  description: 'Portal mahasiswa',
};

// Fungsi layout menerima {children} sebagai prop
export default function RootLayout({ children }) {
  return (
    <html lang="id">
      <body>
        <nav>
          <a href="/">Beranda</a>
          <a href="/about">Tentang</a>
          <a href="/contact">Kontak</a>
        </nav>

        {children}    {/* ← halaman aktif masuk di sini */}

        <footer>© 2024 Website Kampus</footer>
      </body>
    </html>
  );
}

2. Bagaimana {children} Menerima page.js

Ketika kamu membuka halaman /, Next.js otomatis mengambil konten dari app/page.js dan memasukkannya sebagai {children}.

app/page.js
// app/page.js — Halaman utama website
export default function HomePage() {
  return (
    <main>
      <h1>Selamat Datang!</h1>
      <p>Ini adalah halaman utama website kampus.</p>
    </main>
  );
}
Alur penggabungan (ilustrasi)
// layout.js membungkus → page.js
// Jadi {children} di layout menjadi:

<html lang="id">
  <body>
    <nav>...</nav>

    <!-- {children} diganti dengan isi page.js ↓ -->
    <main>
      <h1>Selamat Datang!</h1>
      <p>Ini adalah halaman utama...</p>
    </main>

    <footer>...</footer>
  </body>
</html>

3. Nested Layout untuk Dashboard

Kalau section dashboard punya tampilan khusus (misalnya sidebar), kamu bisa buat layout tambahan di folder app/dashboard/:

app/dashboard/layout.js
// app/dashboard/layout.js — Layout khusus dashboard
// Layout ini dibungkus LAGI oleh Root Layout!

export default function DashboardLayout({ children }) {
  return (
    <div style={{ display: 'flex' }}>
      <aside>
        <h3>Dashboard Menu</h3>
        <a href="/dashboard">Beranda</a>
        <a href="/dashboard/nilai">Nilai</a>
        <a href="/dashboard/jadwal">Jadwal</a>
      </aside>

      <main>
        {children}  {/* ← halaman dashboard aktif */}
      </main>
    </div>
  );
}

4. Membuat Komponen Navbar Terpisah

Best practice: pisahkan navbar ke file komponen sendiri agar layout tetap bersih dan rapi.

components/Navbar.jsx
// components/Navbar.jsx — Komponen navigasi

export default function Navbar() {
  return (
    <nav style={{
      display: 'flex',
      gap: '16px',
      padding: '16px',
      background: '#f8fafc',
    }}>
      <a href="/">Beranda</a>
      <a href="/about">Tentang</a>
      <a href="/contact">Kontak</a>
    </nav>
  );
}
app/layout.js — menggunakan Navbar
// app/layout.js — Import dan gunakan komponen Navbar
import Navbar from '../components/Navbar';

export default function RootLayout({ children }) {
  return (
    <html lang="id">
      <body>
        <Navbar />  {/* ← komponen navbar */}
        {children}
        <footer>© 2024</footer>
      </body>
    </html>
  );
}

5. Metadata Per-Halaman (Override)

Setiap halaman bisa punya metadata sendiri yang meng-override metadata dari layout. Sangat berguna untuk SEO!

app/about/page.js
// app/about/page.js — Halaman ini punya title sendiri

// Metadata ini akan MENGGANTIKAN metadata dari layout
export const metadata = {
  title: 'Tentang Kami — Website Kampus',
  description: 'Halaman tentang website kampus kami',
};

export default function AboutPage() {
  return (
    <main>
      <h1>Tentang Kami</h1>
      <p>Kami adalah platform informasi mahasiswa.</p>
    </main>
  );
}

💡 Dengan cara ini, halaman /about akan punya title "Tentang Kami — Website Kampus" di tab browser, bukan "Website Kampus" dari layout.

⚖️ Perbandingan: Tanpa vs Dengan Layout

Lihat betapa bersihnya kode ketika kamu menggunakan layout dibanding menulis ulang navbar dan footer di setiap halaman!

❌ Tanpa Layout (Repetitif)

Setiap halaman mengulang kode yang sama!

app/page.js
export default function Home() {
  return (
    <>
      <nav>Navbar</nav>  ← duplikat!
      <h1>Beranda</h1>
      <footer>...</footer> ← duplikat!
    </>
  );
}
app/about/page.js
export default function About() {
  return (
    <>
      <nav>Navbar</nav>  ← duplikat!
      <h1>Tentang</h1>
      <footer>...</footer> ← duplikat!
    </>
  );
}
app/contact/page.js
export default function Contact() {
  return (
    <>
      <nav>Navbar</nav>  ← duplikat!
      <h1>Kontak</h1>
      <footer>...</footer> ← duplikat!
    </>
  );
}

😩 Kalau navbar berubah, kamu harus edit SEMUA file!

✅ Dengan Layout (DRY & Bersih)

Layout ditulis sekali, halaman fokus ke konten!

app/layout.js
export default function Layout({ children }) {
  return (
    <html>
      <body>
        <nav>Navbar</nav>  ← sekali saja!
        {children}
        <footer>...</footer> ← sekali saja!
      </body>
    </html>
  );
}
app/page.js
export default function Home() {
  return <h1>Beranda</h1>;
}
app/about/page.js
export default function About() {
  return <h1>Tentang</h1>;
}
app/contact/page.js
export default function Contact() {
  return <h1>Kontak</h1>;
}

😎 Ubah navbar? Cukup edit 1 file!

⚠️ Kesalahan Umum

Berikut adalah 4 kesalahan yang sering dilakukan pemula saat bekerja dengan layout. Pastikan kamu menghindarinya!

❌ Kesalahan 1: Lupa <html> dan <body> di Root Layout

Root layout (app/layout.js) WAJIB mengembalikan tag <html> dan <body>. Tanpa keduanya, aplikasi akan crash dan menampilkan error!

// ❌ SALAH — tidak ada <html> dan <body>
export default function RootLayout({ children }) {
  return <div>{children}</div>;  // CRASH!
}

// ✅ BENAR — wajib ada <html> dan <body>
export default function RootLayout({ children }) {
  return (
    <html><body>{children}</body></html>
  );
}

❌ Kesalahan 2: Import CSS Global di page.js

CSS global (seperti globals.css) harus di-import di layout.js, BUKAN di page.js. Kalau kamu import di page, CSS-nya hanya berlaku ketika halaman itu aktif.

// ❌ SALAH — import CSS global di page.js
// app/page.js
import './globals.css';  // Jangan di sini!

// ✅ BENAR — import CSS global di layout.js
// app/layout.js
import './globals.css';  // Di sini, berlaku global!

❌ Kesalahan 3: Tidak Paham Layout Persist Saat Navigasi

Layout TIDAK di-render ulang ketika user berpindah halaman. Ini artinya state di dalam layout akan tetap tersimpan. Jangan simpan data yang seharusnya reset per-halaman di layout!

Layout tetap "hidup" selama user berada di halaman-halaman yang dibungkusnya. Hanya {children} yang berubah. Ini bagus untuk performa, tapi perlu dipahami agar tidak bingung!

❌ Kesalahan 4: Layout Tanpa {children}

Kalau kamu membuat layout tapi lupa menaruh {children} di dalamnya, halaman-halaman di bawah folder itu tidak akan muncul sama sekali!

// ❌ SALAH — lupa {children}!
export default function Layout({ children }) {
  return (
    <html><body>
      <nav>Navbar</nav>
      {/* Di mana {children}?! Halaman tidak muncul! */}
      <footer>Footer</footer>
    </body></html>
  );
}

// ✅ BENAR — selalu taruh {children}!
export default function Layout({ children }) {
  return (
    <html><body>
      <nav>Navbar</nav>
      {children}  {/* ← WAJIB ada! */}
      <footer>Footer</footer>
    </body></html>
  );
}

✏️ Latihan

Cobalah 3 latihan berikut untuk memperdalam pemahamanmu tentang layout dan children:

Latihan 1 — Root Layout Dasar

Buat file app/layout.js yang berisi:

  • Navbar dengan 3 link: Beranda, Tentang, Kontak
  • Area {children} untuk menampilkan halaman
  • Footer dengan teks copyright
  • Jangan lupa tag <html> dan <body>!

Latihan 2 — Dua Halaman

Buat 2 file halaman yang menggunakan layout dari Latihan 1:

  • app/page.js — Halaman beranda dengan judul dan paragraf sambutan
  • app/about/page.js — Halaman tentang dengan metadata title yang berbeda dari layout

Pastikan kedua halaman otomatis mendapat navbar dan footer dari layout!

Latihan 3 — 🏆 Challenge: Nested Layout Dashboard

Buat nested layout untuk section /dashboard:

  • Buat file app/dashboard/layout.js dengan sidebar berisi 3 link
  • Buat 2 halaman dashboard: app/dashboard/page.js dan app/dashboard/nilai/page.js
  • Pastikan sidebar hanya muncul di halaman /dashboard dan sub-halamannya
  • Halaman /about TIDAK boleh ada sidebar!

📋 Ringkasan

Berikut rangkuman dari semua konsep yang sudah kamu pelajari di halaman ini:

  • ✅ layout.js membungkus semua halaman di folder yang sama dan subfolder-subfoldernya.
  • ✅ Root layout (app/layout.js) WAJIB punya tag <html> dan <body>.
  • ✅ {children} adalah placeholder — isinya adalah halaman yang sedang aktif saat itu.
  • ✅ Layout TIDAK di-render ulang saat navigasi antar halaman — ini membuat perpindahan halaman terasa sangat cepat dan efisien!
  • ✅ CSS yang di-import di layout berlaku secara global untuk semua halaman yang dibungkusnya.
  • ✅ Nested layout = layout di dalam layout. Cocok untuk section seperti dashboard yang punya tampilan berbeda.
  • ✅ Metadata (title, description) yang di-export di halaman akan meng-override metadata dari layout.
