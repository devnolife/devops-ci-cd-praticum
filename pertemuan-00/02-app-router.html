Konsep 02

App Router

Routing otomatis dari struktur folder

🎯 Analogi

🗄️

Bayangkan App Router seperti lemari arsip. Setiap laci (folder) punya label (URL). Buka laci "about" = buka halaman /about. Kamu tidak perlu daftar isi — lokasi file = alamat halaman.

Setiap laci berisi dokumen tertentu bernama page.js. Laci yang ada di dalam laci lain menjadi nested route. Misalnya folder about/team/ berarti URL-nya /about/team. Label di laci = segmen URL. Sesederhana itu!

Dengan analogi ini, kamu bisa memahami bahwa setiap folder di dalam direktori app/ secara otomatis menjadi bagian dari URL. Tidak perlu konfigurasi, tidak perlu install package tambahan. Cukup buat folder dan file — Next.js yang mengurus sisanya.

📖 Apa Itu Routing?

Routing adalah cara browser menentukan halaman mana yang harus ditampilkan ketika kamu mengetik URL tertentu. Pikirkan URL sebagai alamat rumah, dan router sebagai GPS yang mengarahkan kamu ke halaman yang benar.

Ketika kamu mengetik website.com/about, router bertugas mencari halaman "about" dan menampilkannya di layar kamu.

Routing Tradisional vs File-Based

❌ Routing Tradisional (Manual)

Kamu harus menulis konfigurasi sendiri: daftarkan setiap URL, pasangkan dengan komponen, install package react-router-dom, bungkus dengan BrowserRouter. Semakin besar aplikasi, semakin panjang file konfigurasi.

✅ File-Based Routing (Otomatis)

Cukup buat folder dan file page.js di dalamnya. Next.js otomatis mengenali struktur folder sebagai URL. Tidak perlu konfigurasi, tidak perlu package tambahan. Simpel dan bersih!

💡 Tips Di Next.js 13+, sistem routing yang baru disebut App Router. Sistem lama (Pages Router) masih bisa dipakai, tapi App Router adalah cara yang direkomendasikan.

⚙️ Bagaimana Cara Kerjanya?

App Router menggunakan prinsip sederhana: struktur folder menentukan URL. Mari kita pelajari aturan-aturan dasarnya.

Aturan Dasar File-Based Routing

Berikut file-file spesial yang dikenali oleh Next.js App Router:

📄
page.js
Halaman HTML yang bisa dikunjungi lewat URL
🔗
route.js
API endpoint yang mengembalikan data JSON
📐
layout.js
Pembungkus halaman (navbar, footer, dll.)
⏳
loading.js
Tampilan saat halaman sedang dimuat
🚨
error.js
Tampilan saat terjadi error
🔍
not-found.js
Halaman 404 — tidak ditemukan
⚠️ Penting Ingat: folder = path URL, dan hanya file bernama page.js atau route.js yang bisa diakses lewat URL. File lain (komponen, utilitas) tidak akan menjadi halaman.

Dynamic Routes (Rute Dinamis)

Bagaimana kalau kamu punya ratusan halaman blog? Tidak mungkin buat folder satu per satu. Di sinilah dynamic routes bekerja.

Dengan membuat folder bernama [id] (pakai kurung siku), Next.js akan mencocokkan URL apa pun yang ada di segmen itu. Contoh:

  • app/blog/[id]/page.js → cocok dengan /blog/1, /blog/2, /blog/hello
  • Nilai id bisa diakses lewat parameter params

Tabel File Spesial

Ringkasan lengkap file-file yang dikenali oleh App Router:

File Kegunaan Contoh Penggunaan

💻 Contoh Kode

Berikut 5 contoh kode yang sering digunakan saat bekerja dengan App Router. Perhatikan komentar di setiap baris untuk memahami fungsinya.

1. Halaman Utama (page.js)

Ini adalah file paling dasar. Letakkan di app/page.js dan otomatis menjadi halaman utama.

app/page.js
// app/page.js — Halaman utama yang muncul di URL "/"
// File ini WAJIB bernama page.js agar dikenali Next.js

export default function Home() {
  return (
    <main>
      <h1>Selamat Datang!</h1>
      <p>Ini adalah halaman utama website kamu</p>
      <p>Dibuat dengan Next.js App Router ✨</p>
    </main>
  );
}

2. Nested Route (Rute Bersarang)

Untuk membuat halaman /about/team, buat folder about/team/ dan letakkan page.js di dalamnya.

app/about/team/page.js
// app/about/team/page.js — Halaman /about/team
// Folder "about" = segmen /about
// Folder "team"  = segmen /team
// Gabungan = /about/team

export default function Team() {
  return (
    <section>
      <h1>Tim Kami</h1>
      <p>Kenali orang-orang hebat di balik layar</p>
    </section>
  );
}

3. Dynamic Route (Rute Dinamis)

Gunakan folder [id] untuk menangkap URL yang bervariasi. Nilai dari URL bisa diakses lewat properti params.

app/blog/[id]/page.js
// app/blog/[id]/page.js — Halaman dinamis
// Cocok dengan /blog/1, /blog/2, /blog/apapun
// Nilai "id" otomatis ditangkap dari URL

export default function BlogPost({ params }) {
  return (
    <article>
      <h1>Artikel #{params.id}</h1>
      <p>Ini adalah konten artikel dengan ID: {params.id}</p>
    </article>
  );
}
💡 Tips Nama folder di dalam kurung siku menentukan nama parameter. [id] berarti params.id. Kalau kamu buat [slug], maka aksesnya params.slug.

4. API Route (route.js)

File route.js berfungsi sebagai endpoint API. Alih-alih mengembalikan tampilan HTML, file ini mengembalikan data (biasanya JSON).

app/api/hello/route.js
// app/api/hello/route.js — Endpoint API di /api/hello
// File ini TIDAK menampilkan halaman,
// melainkan mengembalikan data JSON

export async function GET() {
  // Response.json() secara otomatis mengatur
  // header Content-Type ke application/json
  return Response.json({
    pesan: "Halo dari API!",
    status: "sukses",
    waktu: new Date().toISOString()
  });
}

5. Custom 404 (not-found.js)

File not-found.js akan otomatis ditampilkan ketika halaman tidak ditemukan. Letakkan di app/not-found.js untuk menangani seluruh aplikasi.

app/not-found.js
// app/not-found.js — Halaman 404 kustom
// Muncul otomatis saat URL tidak cocok
// dengan route manapun

export default function NotFound() {
  return (
    <div style={{ textAlign: 'center', padding: '80px 20px' }}>
      <h1 style={{ fontSize: '4rem' }}>404</h1>
      <p>Halaman yang kamu cari tidak ditemukan 😢</p>
      <a href="/">Kembali ke Beranda</a>
    </div>
  );
}

⚖️ Perbandingan

Mari bandingkan bagaimana routing dilakukan di React biasa (dengan React Router) dan di Next.js App Router. Perbedaannya sangat mencolok!

❌ React Router (Manual)

src/App.jsx
// 1. Install dulu
// npm install react-router-dom

import { BrowserRouter,
  Routes, Route
} from 'react-router-dom';

import Home from './Home';
import About from './About';
import Blog from './Blog';
import Contact from './Contact';

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/"
          element={<Home/>} />
        <Route path="/about"
          element={<About/>} />
        <Route path="/blog"
          element={<Blog/>} />
        <Route path="/contact"
          element={<Contact/>} />
      </Routes>
    </BrowserRouter>
  );
}

⚠️ Harus install package, import setiap halaman, dan daftarkan setiap route secara manual. Semakin banyak halaman = semakin panjang file ini.

✅ Next.js App Router (Otomatis)

Cukup buat folder & file
// Tidak perlu install apa-apa!
// Tidak perlu file konfigurasi!
// Cukup buat struktur folder:

📁 app/
├── page.js         → /
├── about/
│   └── page.js     → /about
├── blog/
│   └── page.js     → /blog
└── contact/
    └── page.js     → /contact

// Selesai! 🎉
// Setiap page.js cukup export
// satu komponen React.

✅ Tidak perlu install package, tidak perlu import, tidak perlu konfigurasi. Buat folder = buat route. Sesederhana itu!

⚠️ Kesalahan Umum

Hindari kesalahan-kesalahan berikut saat menggunakan App Router:

1. Nama File Salah

Di Next.js 13+ (App Router), nama file untuk halaman adalah page.js, bukan index.js.

✗ Salah 
app/about/index.js  ← Tidak dikenali!
✓ Benar 
app/about/page.js   ← Benar! ✅

2. Meletakkan Komponen di Folder app/

File komponen biasa (Button, Card, dll.) sebaiknya diletakkan di folder components/ di luar app/, bukan langsung di dalam folder route.

✗ Salah 
app/Button.js  ← Bukan halaman,
                  jangan taruh di sini!
✓ Benar 
components/Button.js  ← Tempat
                         yang tepat ✅

3. Bingung page.js vs route.js

page.js mengembalikan tampilan UI (HTML). route.js mengembalikan data (JSON). Jangan sampai tertukar!

✗ Salah 
// app/api/users/page.js
// ← Harusnya route.js
//   untuk API!
return Response.json(...)
✓ Benar 
// app/api/users/route.js
// ← route.js untuk API ✅
export async function GET() {
  return Response.json(...)
}

4. Lupa Bahwa Nama Folder = URL

Nama folder bersifat case-sensitive. Folder About dan about menghasilkan URL yang berbeda!

✗ Tidak Konsisten 
app/About/page.js  → /About
app/Blog/page.js   → /Blog
← Huruf besar di URL? 🤔
✓ Konsisten 
app/about/page.js  → /about
app/blog/page.js   → /blog
← Huruf kecil semua ✅
🚫 Perhatian Konvensi umum: gunakan huruf kecil semua (lowercase) untuk nama folder route. Ini mencegah masalah di server Linux yang case-sensitive.

✏️ Latihan

Coba kerjakan latihan-latihan berikut untuk menguji pemahamanmu:

1

Buat 3 halaman: Buat proyek Next.js baru dan buat tiga halaman — / (beranda), /about (tentang), dan /contact (kontak). Setiap halaman harus menampilkan judul dan deskripsi singkat. Tulis struktur folder yang kamu buat.

2

Dynamic route blog: Buat dynamic route /blog/[id] yang menampilkan nomor artikel berdasarkan ID dari URL. Misalnya, kunjungi /blog/5 harus menampilkan "Artikel #5". Gunakan params.id untuk mengambil nilainya.

3

API endpoint: Buat API endpoint di /api/mahasiswa yang mengembalikan daftar nama mahasiswa dalam format JSON. Gunakan route.js dengan fungsi GET. Contoh response: { "mahasiswa": ["Andi", "Budi", "Citra"] }.

💡 Petunjuk Untuk latihan 1, kamu hanya perlu membuat 3 folder (root sudah ada) dan 3 file page.js. Untuk latihan 3, ingat: API menggunakan route.js, bukan page.js!

📋 Ringkasan

Berikut poin-poin penting yang telah kamu pelajari tentang App Router:

  • ✅ Folder di app/ = path URL — buat folder, otomatis jadi route
  • ✅ page.js = halaman yang bisa dikunjungi pengguna lewat browser
  • ✅ route.js = API endpoint yang mengembalikan data (biasanya JSON)
  • ✅ [nama] = dynamic route — menangkap URL yang bervariasi
  • ✅ layout.js membungkus semua halaman dalam satu folder (dan sub-folder)
  • ✅ loading.js & error.js untuk pengalaman pengguna yang lebih baik
  • ✅ Tidak perlu install react-router — semuanya sudah built-in di Next.js!
📌 Ingat Kunci utama App Router: struktur folder = struktur URL. Kalau kamu bisa membuat folder dan file, kamu sudah bisa membuat routing di Next.js!
