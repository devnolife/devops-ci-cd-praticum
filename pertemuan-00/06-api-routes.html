API Routes

Backend di dalam Next.js — satu gedung, dua fungsi

🎯 Analogi Sederhana

API Route itu seperti jendela drive-through restoran.

Kamu (browser) pesan dari luar: "Saya mau data health!" — server di dalam dapur menyiapkan data dan memberikannya lewat jendela dalam format JSON.

Satu gedung restoran (Next.js) punya dua bagian:
🍽️ Ruang makan = frontend (halaman yang dilihat user)
👨‍🍳 Dapur = backend (API yang mengolah data)

Bayangkan HTTP methods seperti ini:
📖 GET = baca menu  ·  ✏️ POST = pesan makanan  ·  🔄 PUT = ubah pesanan  ·  🗑️ DELETE = batalkan pesanan

📖 Apa itu API?

API = Application Programming Interface — cara aplikasi bicara satu sama lain. Sama seperti kamu pesan makanan lewat kasir (interface), bukan langsung masuk ke dapur.

Alur Kerja API

Browser mengirim REQUEST → Server memproses → Server mengirim RESPONSE.

Format Data: JSON

Data dikirim dalam format JSON (JavaScript Object Notation) — mirip objek JavaScript tapi bisa dibaca oleh bahasa pemrograman apa pun.

contoh-data.json
{
  "nama": "Budi Santoso",
  "nim": "2024001",
  "jurusan": "Teknik Informatika",
  "aktif": true
}

RESTful API

REST (Representational State Transfer) adalah gaya arsitektur yang menggunakan HTTP methods untuk operasi data:

  • GET — mengambil data
  • POST — mengirim data baru
  • PUT — mengupdate data yang sudah ada
  • DELETE — menghapus data
💡 Tips Kamu bisa menguji API menggunakan browser (untuk GET), atau tools seperti Postman dan Thunder Client (VS Code extension) untuk semua methods.

⚙️ API di Next.js

Di Next.js App Router, kamu membuat API endpoint dengan file bernama route.js (bukan page.js!). File ini diletakkan di folder app/api/.

Aturan Dasar

  • Buat file route.js di dalam app/api/NAMA/
  • Export fungsi sesuai HTTP method: GET, POST, PUT, DELETE
  • Kembalikan response dengan NextResponse.json({...})
  • Endpoint otomatis tersedia di /api/NAMA
💡 Ingat Folder yang sama TIDAK boleh punya page.js dan route.js sekaligus. Pilih salah satu — halaman atau API.

📬 HTTP Methods

Setiap HTTP method punya tugas berbeda. Berikut penjelasannya dengan analogi yang mudah dipahami:

📖
GET
Ambil data — seperti baca buku di perpustakaan
✏️
POST
Kirim data baru — seperti mengisi formulir pendaftaran
🔄

Update data — seperti mengedit dokumen yang sudah ada

💡 Tips Browser secara default menggunakan GET ketika kamu membuka URL. Untuk mengirim POST, PUT, atau DELETE, kamu harus menggunakan fetch() atau tools seperti Postman.

💻 Contoh Kode

Berikut contoh-contoh API route yang bisa langsung kamu praktikkan:

1. Simple GET — Health Check

Endpoint paling dasar: mengembalikan status server.

app/api/health/route.js
// Import NextResponse dari next/server
import { NextResponse } from 'next/server'

// Export fungsi GET — otomatis jadi endpoint GET
export async function GET() {
  // Kembalikan data dalam format JSON
  return NextResponse.json({
    status: "healthy",
    timestamp: new Date().toISOString(),
    uptime: process.uptime()
  })
}

2. POST — Menerima Data

Endpoint yang menerima data dari request body.

app/api/feedback/route.js
import { NextResponse } from 'next/server'

// Fungsi POST — menerima data dari body request
export async function POST(request) {
  // Ambil data JSON dari body request
  const body = await request.json()

  // Proses data (simpan ke database, dll)
  console.log('Data diterima:', body)

  // Kirim response sukses
  return NextResponse.json({
    success: true,
    data: body
  })
}

3. Query Parameters — Data Dinamis

Membaca parameter dari URL, misalnya /api/greet?name=Budi.

app/api/greet/route.js
import { NextResponse } from 'next/server'

export async function GET(request) {
  // Ambil search params dari URL
  const { searchParams } = new URL(request.url)
  const name = searchParams.get('name') || 'World'

  // Kirim greeting dengan nama dari parameter
  return NextResponse.json({
    message: `Halo, ${name}! Selamat datang 👋`
  })
}

4. Error Handling — Menangani Kesalahan

Selalu bungkus logic API kamu dengan try-catch supaya error tidak crash.

app/api/data/route.js
import { NextResponse } from 'next/server'

export async function GET() {
  try {
    // Coba ambil data (bisa dari database, API luar, dll)
    const data = await fetchData()
    return NextResponse.json(data)
  } catch (error) {
    // Kalau gagal, kirim pesan error dengan status 500
    return NextResponse.json(
      { error: 'Terjadi kesalahan pada server' },
      { status: 500 }
    )
  }
}

5. Memanggil API dari Frontend

Gunakan fetch() untuk memanggil API route dari komponen React.

app/dashboard/page.js
'use client'

import { useState, useEffect } from 'react'

export default function Dashboard() {
  const [data, setData] = useState(null)

  async function getData() {
    // Panggil API route kita sendiri
    const res = await fetch('/api/health')
    const json = await res.json()
    setData(json)
  }

  useEffect(() => { getData() }, [])

  return <div>{data?.status}</div>
}

📦 Request & Response

Objek Request

Ketika API route menerima request, objek request berisi informasi berikut:

  • url — alamat lengkap yang dipanggil
  • method — HTTP method (GET, POST, dll)
  • headers — informasi tambahan (Content-Type, Authorization, dll)
  • body — data yang dikirim (untuk POST/PUT)

Response & Status Codes

Server mengirim response dengan status code yang menunjukkan hasil operasi:

200 OK — Berhasil
201 Created — Data dibuat
400 Bad Request — Salah input
404 Not Found — Tidak ditemukan
500 Server Error — Rusak di server
contoh-response-dengan-status.js
// Response berhasil (default status 200)
return NextResponse.json({ data: "sukses" })

// Response data baru dibuat (status 201)
return NextResponse.json(
  { message: "User berhasil dibuat" },
  { status: 201 }
)

// Response error — data tidak ditemukan (status 404)
return NextResponse.json(
  { error: "User tidak ditemukan" },
  { status: 404 }
)

⚖️ Perbandingan: page.js vs route.js

Banyak pemula bingung kapan pakai page.js dan kapan pakai route.js. Berikut perbandingannya:

Aspek page.js route.js
Fungsi Halaman UI yang dilihat user API endpoint untuk data
Return JSX / HTML JSON
Lokasi app/nama/ app/api/nama/
Akses Browser langsung fetch() / Postman
Contoh URL /about /api/health
Render Server / Client Component Tidak di-render — hanya data
⚠️ Penting Satu folder TIDAK boleh punya page.js dan route.js bersamaan. Next.js akan error kalau kamu melakukan itu. Pilih salah satu!

⚠️ Kesalahan Umum

Berikut 4 kesalahan yang sering dilakukan pemula saat membuat API routes:

1. Menggunakan page.js untuk API

❌ Salah
app/api/health/page.js
// ❌ Ini salah! page.js untuk halaman UI, bukan API
export default function Health() {
  return <p>Healthy</p>
}
✅ Benar
app/api/health/route.js
// ✅ Gunakan route.js dan return JSON
import { NextResponse } from 'next/server'

export async function GET() {
  return NextResponse.json({ status: "healthy" })
}

2. Lupa NextResponse.json()

❌ Salah
route.js
// ❌ Return object biasa — browser tidak bisa baca ini!
export async function GET() {
  return { status: "ok" }
}
✅ Benar
route.js
// ✅ Bungkus dengan NextResponse.json()
import { NextResponse } from 'next/server'

export async function GET() {
  return NextResponse.json({ status: "ok" })
}

3. Tidak Handle Error

❌ Salah
route.js
// ❌ Kalau fetchData() gagal, server langsung crash!
export async function GET() {
  const data = await fetchData()
  return NextResponse.json(data)
}
✅ Benar
route.js
// ✅ Bungkus dengan try-catch supaya aman
export async function GET() {
  try {
    const data = await fetchData()
    return NextResponse.json(data)
  } catch (error) {
    return NextResponse.json(
      { error: 'Server error' },
      { status: 500 }
    )
  }
}

4. File di Lokasi Salah

❌ Salah
app/health/route.js ← Bukan di folder api/!
// ❌ File ini akan jadi route /health bukan /api/health
// Bisa bentrok dengan halaman page.js lainnya
export async function GET() {
  return NextResponse.json({ status: "ok" })
}
✅ Benar
app/api/health/route.js ← Di dalam folder api/
// ✅ Letakkan di app/api/ agar rapi dan jelas
import { NextResponse } from 'next/server'

export async function GET() {
  return NextResponse.json({ status: "ok" })
}

✏️ Latihan

Sekarang giliran kamu! Coba kerjakan latihan-latihan berikut:

Latihan 1 — Endpoint Profil

Buat endpoint GET /api/profile yang mengembalikan data JSON berisi nama, NIM, dan jurusan kamu. Buat file di app/api/profile/route.js.

Hint: Export fungsi GET dan return NextResponse.json({...}) dengan data profilmu.

Latihan 2 — Greeting Dinamis

Buat endpoint GET /api/greet?name=Budi yang membaca parameter name dari URL dan mengembalikan pesan "Halo, Budi! Selamat belajar Cloud Native 🚀". Kalau name tidak diisi, gunakan default "Teman".

Hint: Gunakan new URL(request.url).searchParams.get('name')

Latihan 3 — POST Feedback

Buat endpoint POST /api/feedback yang menerima JSON body berisi nama dan pesan, lalu mengembalikan response:

{
  "success": true,
  "diterima": {
    "nama": "Budi",
    "pesan": "Materinya seru!"
  },
  "waktu": "2024-01-15T10:30:00.000Z"
}

Hint: Gunakan await request.json() untuk membaca body dan tambahkan new Date().toISOString() untuk timestamp.

📋 Ringkasan

Berikut poin-poin penting yang sudah kamu pelajari di halaman ini:

  • ✅ API = cara aplikasi bicara satu sama lain lewat HTTP
  • ✅ Di Next.js, gunakan file route.js di folder app/api/
  • ✅ Export fungsi sesuai HTTP method: GET, POST, PUT, DELETE
  • ✅ Return data dengan NextResponse.json() dari next/server
  • ✅ Panggil API dari frontend dengan fetch('/api/nama')
  • ✅ Selalu handle error dengan try-catch supaya server tidak crash
  • ✅ Jangan campur page.js dan route.js di folder yang sama
  • ✅ Gunakan status code yang tepat: 200 (ok), 201 (created), 400 (bad request), 404 (not found), 500 (error)
