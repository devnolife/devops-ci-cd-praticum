Build & Docker

Dari kode ke container — siap deploy ke mana saja

🎯 Analogi

🍱

Build itu seperti memasak makanan beku. Kamu masak (build) sekali di dapur, lalu bekukan hasilnya jadi Docker image. Siapa saja bisa panaskan (docker run) dan makan di mana saja — hasilnya PERSIS sama.

  • 🐳 Docker = kotak makan beku yang menjamin makanan sama di mana pun dipanaskan.
  • 📝 Dockerfile = resep memasak. Ikuti langkah-langkahnya, hasilnya selalu sama.
  • ⚡ Multi-stage build = masak di dapur besar, tapi hanya bawa makanan jadi ke meja makan — bukan semua alat masak.
💡 Inti Masalah yang Diselesaikan Docker "Di laptop saya jalan kok!" — kalimat ini hilang selamanya kalau kamu pakai Docker. Semua orang menjalankan environment yang 100% identik.

📖 Build Process

Sebelum masuk Docker, kamu harus paham dulu proses build aplikasi Next.js. Ini langkah-langkahnya:

  1. npm install — download semua bahan (dependencies) yang dibutuhkan
  2. npm test — cek kualitas kode (testing), pastikan tidak ada yang rusak
  3. npm run build — "masak" kode jadi produk jadi (folder .next/)
  4. output: "standalone" di next.config.js — hasilkan server.js yang minimal dan mandiri

Konfigurasi next.config.js

next.config.js
/** @type {import('next').NextConfig} */
const nextConfig = {
  output: "standalone",
  // Menghasilkan server.js mandiri
  // Ukuran jauh lebih kecil dari full node_modules
};

module.exports = nextConfig;

Apa isi folder .next/ setelah build?

  • 📄 HTML files — halaman yang sudah di-render
  • 📦 JS chunks — kode JavaScript yang sudah di-split dan di-optimize
  • 🖥️ server.js — file server mandiri (kalau pakai standalone output)
  • 🎨 static/ — file CSS, gambar, dan aset lainnya
⚠️ Penting! Tanpa output: "standalone", kamu butuh seluruh node_modules untuk menjalankan server. Dengan standalone, hanya perlu file server.js yang kecil (~15 MB vs 300+ MB).

🐳 Konsep Docker

Sebelum menulis Dockerfile, pahami dulu istilah-istilah kunci ini:

💿 Image

Blueprint / cetakan (read-only). Seperti master CD — tidak bisa diubah, tapi bisa di-copy berkali-kali.

📦 Container

Instance yang jalan dari sebuah image. Seperti CD yang sedang diputar di player — bisa ada banyak player sekaligus.

📝 Dockerfile

Resep untuk membuat image. Berisi instruksi step-by-step, dari base image sampai perintah terakhir.

🏪 Registry

Tempat simpan image (Docker Hub). Seperti toko CD — kamu push image ke sana, orang lain bisa pull.

⚡ Multi-stage Build

Beberapa tahap build dalam satu Dockerfile. Hanya hasil akhir yang masuk image final. Bayangkan masak di dapur besar (banyak alat), tapi yang dibawa ke meja makan hanya piring berisi makanan jadi.

🔄 Build Flow

Alur lengkap dari kode sampai container berjalan:

💻 Dockerfile Lengkap

Berikut Dockerfile multi-stage lengkap untuk Next.js. Setiap baris sudah diberi penjelasan:

Dockerfile
# === Stage 1: BASE ===
FROM node:20-alpine AS base
# Pakai Node.js 20 versi ringan (alpine ~5 MB)

# === Stage 2: DEPENDENCIES ===
FROM base AS deps
WORKDIR /app
# Set folder kerja di /app
COPY package*.json ./
# Copy file package.json dan package-lock.json
RUN npm ci --only=production
# Install dependencies (versi exact dari lock file)

# === Stage 3: BUILDER ===
FROM base AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
# Copy node_modules dari stage deps
COPY . .
# Copy semua source code
RUN npm run build
# Build aplikasi Next.js → .next/ folder

# === Stage 4: RUNNER ===
FROM base AS runner
WORKDIR /app
ENV NODE_ENV=production
# Set mode production

RUN addgroup --system --gid 1001 nodejs
RUN adduser --system --uid 1001 nextjs
# Buat user non-root (keamanan!)

COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static
COPY --from=builder /app/public ./public
# Copy HANYA file yang dibutuhkan

USER nextjs
# Jalankan sebagai user non-root
EXPOSE 3000
CMD ["node", "server.js"]
# Jalankan server
🔒 Kenapa pakai USER non-root? Kalau container dijalankan sebagai root dan ada celah keamanan, attacker bisa dapat akses root ke seluruh sistem host. Dengan user biasa, dampak serangan jauh lebih terbatas.

🚫 File .dockerignore

Sama seperti .gitignore, file ini memberitahu Docker file apa yang TIDAK perlu di-copy ke dalam image:

.dockerignore
node_modules
.next
.git
*.md
.env*.local

Kenapa setiap entry penting?

  • node_modules — akan di-install ulang di dalam Docker, tidak perlu copy dari lokal (bisa 500+ MB!)
  • .next — hasil build lokal kamu, akan di-build ulang di Docker
  • .git — history Git tidak dibutuhkan di production (bisa ratusan MB)
  • *.md — file dokumentasi tidak perlu di dalam container
  • .env*.local — file environment lokal, JANGAN sampai masuk image (bahaya keamanan!)
⚠️ Tanpa .dockerignore Docker akan copy SEMUA file di direktori kamu ke dalam image, termasuk node_modules, .git, dan file rahasia. Image bisa membengkak 5-10x lipat!

🛠️ Docker Commands

Perintah-perintah Docker yang paling sering kamu gunakan:

Build Image

Terminal
# Build image dari Dockerfile di direktori saat ini
docker build -t nextjs-app .

# -t = tag/nama image
# . = gunakan Dockerfile di folder ini

Run Container

Terminal
# Jalankan container
docker run -p 3000:3000 nextjs-app

# Jalankan dengan environment variables
docker run -e NODE_ENV=production -p 3000:3000 nextjs-app

# -p 3000:3000 = port host:container
# -e = set environment variable

Kelola Container & Image

Terminal
# Lihat semua image
docker images

# Lihat container yang sedang berjalan
docker ps

# Stop container
docker stop <container-id>

# Hapus container
docker rm <container-id>

# Hapus image
docker rmi nextjs-app

📦 Docker Compose

Docker Compose memungkinkan kamu mendefinisikan dan menjalankan aplikasi multi-container dengan satu file konfigurasi:

docker-compose.yaml
version: '3.8'
services:
  app:
    build: .
    # Build dari Dockerfile di folder ini
    ports:
      - "3000:3000"
    # Map port 3000 host ke 3000 container
    environment:
      - NODE_ENV=production
      - APP_MESSAGE=Hello from Compose!
    # Environment variables
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/api/health"]
      interval: 30s
      # Cek kesehatan setiap 30 detik
      timeout: 10s
      # Timeout 10 detik per check
      retries: 3
      # Coba 3 kali sebelum dianggap unhealthy

Perintah Docker Compose

Terminal
# Build dan jalankan semua services
docker compose up --build

# Jalankan di background
docker compose up -d

# Stop semua services
docker compose down
💡 Kapan pakai Docker Compose? Gunakan Docker Compose kalau aplikasi kamu punya lebih dari satu service (misal: app + database + redis). Untuk satu container saja, docker run sudah cukup.

⚖️ Perbandingan

Apa bedanya development tanpa dan dengan Docker?

Aspek Tanpa Docker 😰 Dengan Docker 🐳
Setup Install Node, npm, dll satu-satu di setiap mesin docker run saja — semua sudah termasuk
Konsistensi "Di laptop saya jalan kok!" 😅 Pasti sama di mana pun — development, staging, production
Deploy Manual, ribet, banyak langkah docker push + docker pull — selesai
Isolasi Bisa bentrok dengan app lain di server yang sama Terisolasi sempurna, tidak saling ganggu
Ukuran Semua file terbawa (termasuk devDependencies) Hanya file yang dibutuhkan (multi-stage build)

⚠️ Kesalahan Umum

Hindari 4 kesalahan ini saat menggunakan Docker:

❌ 1. Tidak pakai .dockerignore

Image jadi sangat besar karena node_modules, .git, dan file tidak perlu lainnya ikut ter-copy. Image yang seharusnya 150 MB bisa jadi 1.5 GB!

✅ Selalu buat file .dockerignore sebelum docker build

❌ 2. Tidak pakai multi-stage build

Semua node_modules (termasuk devDependencies), source code, dan file build masuk ke image final. Boros ruang dan lambat di-deploy.

✅ Gunakan multi-stage build — hanya copy hasil akhir ke image runner

❌ 3. Jalankan sebagai root di container

Kalau ada celah keamanan, attacker bisa dapat akses root. Ini sangat berbahaya di production!

✅ Buat user non-root dengan adduser dan gunakan USER directive

❌ 4. Lupa set NODE_ENV=production

Tanpa NODE_ENV=production, Next.js jalan di mode development — lebih lambat, banyak logging, dan tidak aman.

✅ Selalu set ENV NODE_ENV=production di Dockerfile

✏️ Latihan

Coba kerjakan latihan berikut untuk mempraktikkan apa yang sudah kamu pelajari:

Latihan 1: Dockerfile Sederhana

Tulis Dockerfile tanpa multi-stage untuk Next.js app. Gunakan node:20-alpine sebagai base image, copy semua file, install dependencies, build, dan jalankan.

Latihan 2: Build & Run

Build Docker image dari Dockerfile yang sudah kamu tulis, lalu jalankan container-nya. Buka browser dan akses http://localhost:3000. Pastikan aplikasi berjalan dengan benar.

Latihan 3: Docker Compose + Healthcheck

Buat file docker-compose.yaml yang menjalankan aplikasi kamu. Tambahkan healthcheck yang mengecek endpoint /api/health setiap 30 detik. Jalankan dengan docker compose up dan verifikasi healthcheck bekerja.

💡 Tips mengerjakan latihan Mulai dari Latihan 1 dulu. Kalau sudah berhasil, baru lanjut ke Latihan 2 dan 3. Jangan lupa buat file .dockerignore sebelum build!

Checklist yang harus kamu ingat dari materi ini:

  • ✅ Build: npm install → npm test → npm run build (urutan wajib!)
  • ✅ Docker Image = blueprint (read-only), Container = instance yang sedang berjalan
  • ✅ Dockerfile = resep untuk membuat image, step-by-step
  • ✅ Multi-stage build: dari ~1.2 GB → ~150 MB (hemat 87%!)
  • ✅ Selalu pakai .dockerignore — hindari image yang membengkak
  • ✅ JANGAN jalankan sebagai root — buat user non-root untuk keamanan
  • ✅ Set NODE_ENV=production — performa dan keamanan lebih baik
