● Konsep 01

React vs Next.js

React adalah library UI. Next.js adalah framework fullstack yang membungkus React dengan routing, API, dan build tools.

⚛️ React

Library untuk membangun UI

Hanya UI — tanpa routing, tanpa API
// Butuh install react-router
// Butuh setup Webpack sendiri
// Tidak ada API backend
// Tidak ada SSR built-in

import { BrowserRouter } from 'react-router-dom'
// ^ harus install manual

▲ Next.js

Framework fullstack = React + segalanya

Routing otomatis + API + Build
// ✅ Routing dari folder
// ✅ API route built-in
// ✅ SSR & SSG built-in
// ✅ next build → production

// Buat file → jadi halaman!
app/about/page.js → /about

● Konsep 04

Component, Props & JSX

Component adalah fungsi yang mengembalikan UI. Props adalah data yang dikirim dari parent ke child.

Component Tree — Data mengalir ke bawah via Props

HTML Biasa

Yang sudah kamu tahu

<div class="hero">
  <h1>Judul</h1>
  <p>Teks statis</p>
</div>

JSX (di Next.js)

HTML + JavaScript = dinamis!

<div className="hero">
  <h1>{title}</h1>        // variabel
  <p>{message}</p>      // dari env
  <MetricCard value="UI" /> // component
</div>

● Konsep 05

CSS — Variables, Flexbox, Grid, Responsive

4 konsep CSS utama yang dipakai di aplikasi ini.

🎨

CSS Variables

Definisi warna di :root, pakai di mana saja dengan var(--nama). Ubah di satu tempat → berubah di mana-mana.

/* Definisi */
:root { --brand: #38bdf8; }

/* Pakai */
.hero { color: var(--brand); }
↔️

Flexbox — Layout 1D

Menyusun item dalam satu baris (horizontal) atau satu kolom (vertikal).

A
B
C
D

display: flex; gap: 8px;

⊞

CSS Grid — Layout 2D

Menyusun item dalam baris DAN kolom sekaligus. Cocok untuk kartu, dashboard.

A
B
C
D
E
F

display: grid; grid-template-columns: repeat(3, 1fr);

📱

Responsive — @media

Aturan CSS yang hanya aktif di ukuran layar tertentu. Desktop → 3 kolom, mobile → 1 kolom.

/* Desktop */
.metrics { grid-template-columns:
  repeat(3, 1fr); }

/* Mobile (≤900px) */
@media (max-width: 900px) {
  .metrics { grid-template-columns: 1fr; }
}

● Konsep 07

Environment Variable

Konfigurasi yang bisa diubah TANPA mengubah kode. Satu kode, banyak environment.

Satu kode, tiga environment berbeda

✅ NEXT_PUBLIC_*

Dibaca di browser + server

// Aman untuk data non-sensitif
NEXT_PUBLIC_APP_MESSAGE="Halo!"
NEXT_PUBLIC_DOCKER_IMAGE="app:v1"

🔒 Tanpa Prefix

HANYA dibaca di server

// Untuk secret & konfigurasi server
APP_ENV=production
APP_VERSION=1.0.0
DATABASE_URL=postgres://...  // aman!
