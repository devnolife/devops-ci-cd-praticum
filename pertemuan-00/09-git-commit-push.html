Panduan Git

Commit & Push Tugas

Panduan lengkap cara menyimpan dan mengirim pekerjaan kamu ke GitHub. Gunakan halaman ini setiap kali selesai mengerjakan tugas.

🤔 Kenapa Harus Git?

⏪

Git itu seperti CTRL+Z tapi lebih canggih. Bisa simpan versi kode, lihat perubahan, dan kembali ke versi lama kapan saja. Setiap "snapshot" disimpan selamanya, jadi kamu tidak akan pernah kehilangan kode.

Alur Kerja Git

Git bekerja dalam 4 area utama. Pahami alur ini, dan kamu akan paham Git selamanya:

💡 Ingat Alur Ini git add → git commit → git push. Tiga perintah ini yang akan kamu gunakan setiap hari. Mudah!

⚙️ Setup Git (Pertama Kali Saja)

Langkah ini hanya perlu dilakukan sekali saja di laptop kamu. Setelah setup, tidak perlu diulang lagi.

1. Cek apakah Git sudah terinstall

Terminal
git --version

Jika muncul versi (misal git version 2.43.0), berarti Git sudah terinstall. Jika belum, download dari git-scm.com.

2. Setup nama dan email

Terminal
git config --global user.name "Nama Kamu"
git config --global user.email "email@mahasiswa.ac.id"
💡 Gunakan email yang terdaftar di GitHub Email ini akan muncul di setiap commit kamu. Pastikan sama dengan email akun GitHub kamu agar commit tercatat dengan benar.

3. Verifikasi konfigurasi

Terminal
git config --global --list

📝 Alur Kerja Git (Setiap Kali Selesai Tugas)

Ini adalah bagian paling penting. Ikuti 4 langkah ini setiap kali kamu selesai mengerjakan tugas:

1

Cek Status

Lihat file apa saja yang berubah sejak commit terakhir.

Terminal
git status

File berwarna merah = belum di-add. File berwarna hijau = sudah di-staging.

2

Tambahkan File ke Staging

Tambahkan semua file yang berubah ke staging area.

Terminal
git add .

Tanda titik (.) artinya "semua file". Kamu juga bisa menambahkan file tertentu saja: git add nama-file.js

3

Commit (Simpan Perubahan)

Simpan snapshot perubahan kamu dengan pesan yang deskriptif.

Terminal
git commit -m "pesan commit kamu di sini"

Pesan commit harus menjelaskan apa yang berubah. Lihat contoh pesan commit di bagian selanjutnya.

4

Push (Kirim ke GitHub)

Kirim semua commit kamu ke GitHub agar dosen bisa melihat pekerjaan kamu.

Terminal
git push origin main

Setelah push berhasil, buka GitHub di browser untuk memastikan file kamu sudah muncul.

⚠️ Jangan Lupa Push! Commit saja TIDAK cukup. Kalau hanya commit tanpa push, kode kamu hanya tersimpan di laptop — dosen tidak bisa melihatnya di GitHub.

💬 Format Pesan Commit yang Baik

Pesan commit yang baik membantu kamu dan dosen memahami apa yang berubah. Gunakan format ini:

Template per Pertemuan

Pertemuan Contoh Pesan Commit
Pertemuan 00feat: setup proyek nextjs
Pertemuan 01feat: tambah halaman about dan contact
Pertemuan 02feat: implementasi dynamic routing products
Pertemuan 03feat: buat layout dengan navbar dan footer
Pertemuan 04feat: buat komponen KartuMahasiswa dengan props
Pertemuan 05style: tambah CSS module untuk halaman utama
Pertemuan 06feat: buat API route /api/mahasiswa
Pertemuan 07feat: setup environment variables
Pertemuan 08feat: buat Dockerfile untuk deployment

Contoh Baik vs Buruk

❌ "update" — terlalu vague, update apa?
❌ "fix" — fix apa? Di mana?
❌ "asdfgh" — ini bukan pesan commit
✅ "feat: tambah halaman profil mahasiswa"
✅ "fix: perbaiki error di API route users"
✅ "style: update warna navbar menjadi biru"
💡 Format Commit Message Gunakan pola: tipe: deskripsi singkat. Tipe yang umum: feat (fitur baru), fix (perbaikan bug), style (perubahan tampilan), docs (dokumentasi).

🎓 Cara Submit Tugas di GitHub Classroom

Dosen menggunakan GitHub Classroom untuk membagikan dan mengumpulkan tugas. Ikuti langkah berikut:

1

Buka Link dari Dosen

Dosen akan membagikan link GitHub Classroom untuk setiap tugas. Klik link tersebut.

2

Accept Assignment

Klik tombol "Accept this assignment". GitHub otomatis membuat repository khusus untuk kamu.

3

Clone Repository

Copy URL repository lalu jalankan:

Terminal
git clone <url-dari-classroom>
4

Kerjakan Tugas

Masuk ke folder yang sudah di-clone, lalu kerjakan tugas di dalamnya.

Terminal
cd nama-folder-tugas
5

Add, Commit, Push

Setelah selesai, jalankan tiga perintah ajaib:

Terminal
git add .
git commit -m "feat: selesaikan tugas pertemuan X"
git push origin main
6

Verifikasi di GitHub

Buka repository di browser GitHub. Pastikan semua file kamu sudah muncul dan commit terlihat.

💡 Tips Penting Setelah push, cek di browser GitHub bahwa file kamu sudah muncul. Kalau belum, coba git push lagi. Pastikan juga kamu berada di folder yang benar saat menjalankan perintah Git.

🔥 Perintah Git yang Sering Dipakai

Referensi cepat — klik kartu untuk menyalin perintah:

git status

Cek file yang berubah

git add .

Tambah semua file

git add <file>

Tambah file tertentu

git commit -m "..."

Simpan perubahan

git push

Kirim ke GitHub

git pull

Ambil perubahan terbaru

git log --oneline

Lihat riwayat commit

git diff

Lihat perubahan yang belum di-add

⚠️ Troubleshooting Git

Masalah umum dan cara mengatasinya:

fatal: not a git repository
Penyebab: Kamu menjalankan perintah Git di luar folder proyek.
Solusi: Pastikan kamu sudah cd ke folder proyek. Jika folder belum di-init, jalankan git init.
error: failed to push some refs
Penyebab: Ada perubahan di GitHub yang belum kamu ambil ke lokal.
Solusi: Jalankan git pull origin main dulu, lalu git push origin main lagi.
Permission denied (publickey)
Penyebab: SSH key belum di-setup atau sudah expired.
Solusi: Setup SSH key atau gunakan HTTPS URL + personal access token. Cek panduan di docs.github.com.
CONFLICT (content): Merge conflict in file.js
Penyebab: Dua orang mengubah baris yang sama di file yang sama.
Solusi: Buka file yang conflict, cari tanda <<<<<<<, pilih versi yang benar, hapus tanda conflict, lalu git add . dan git commit.
⚠️ Jangan Panik! Error di Git itu normal. Baca pesan error-nya baik-baik — Git biasanya memberitahu apa yang harus dilakukan. Kalau masih bingung, tanya dosen atau asisten.
