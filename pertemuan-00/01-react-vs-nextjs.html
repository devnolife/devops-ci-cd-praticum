Konsep 01

React vs Next.js

Memahami perbedaan library dan framework — kapan harus pakai yang mana, dan kenapa Next.js jadi standar baru.

🎯 Analogi Sederhana

🍳

Bayangkan kamu mau masak kue.

React = bahan masakan (tepung, telur, gula, mentega). Kamu punya semua bahan yang dibutuhkan, tapi kamu harus tahu sendiri urutan mencampur, suhu oven, dan berapa lama memanggangnya.

Next.js = resep masak lengkap yang sudah mengatur semua bahan tadi. Resepnya bilang: "campurkan tepung dan gula dulu, lalu tambahkan telur, panggang di 180°C selama 30 menit." Hasilnya? Konsisten dan enak setiap kali.

Kamu bisa masak tanpa resep — tapi hasilnya tidak konsisten. Kadang gosong, kadang kurang matang. Sama seperti React biasa: kamu bisa bikin aplikasi web, tapi harus atur sendiri routing, build system, dan banyak hal lainnya.

💡 Intinya: React menyediakan bahan (komponen, hooks, JSX) — Next.js menyediakan resep lengkap (struktur folder, routing otomatis, build process, dan cara deploy).

📖 Apa Itu React & Next.js?

⚛️ React — Library JavaScript untuk UI

React adalah library (bukan framework!) yang dibuat oleh Meta (Facebook). Tugasnya cuma satu: membuat tampilan (User Interface). React menggunakan pendekatan component-based — kamu memecah halaman menjadi potongan-potongan kecil yang bisa dipakai ulang.

  • Hanya mengurus tampilan (view layer)
  • Tidak punya routing bawaan — harus install sendiri
  • Tidak punya backend — butuh server terpisah
  • Butuh setup build tool (Webpack/Vite) secara manual

▲ Next.js — Framework Fullstack

Next.js adalah framework yang membungkus React. Dibuat oleh Vercel. Dia menambahkan semua yang React tidak punya:

  • Routing otomatis — buat file = buat halaman
  • API backend — bisa buat endpoint langsung
  • Build system — next build langsung jalan
  • SSR & SSG — rendering di server, SEO lebih bagus
  • Deployment mudah — deploy ke Vercel dengan 1 klik

Diagram: React berada di dalam Next.js. Panah menunjukkan fitur yang ditambahkan Next.js.

⚙️ Bagaimana Cara Kerjanya?

Mari kita bandingkan fitur-fitur utama antara React biasa dan Next.js:

Fitur React Biasa Next.js
Routing ✗ Install react-router manual, konfigurasi sendiri ✓ Otomatis dari struktur folder
Backend API ✗ Butuh Express.js atau server terpisah ✓ Built-in via route.js
Build ✗ Setup Webpack / Vite sendiri ✓ next build otomatis
SSR ✗ Harus setup manual yang rumit ✓ Default — sudah aktif
CSS Support ✗ Import manual, config loader sendiri ✓ Global CSS + CSS Modules built-in
Deployment ✗ Kompleks, banyak konfigurasi ✓ Vercel 1-click atau Docker
⚠️ Penting! React biasa bukan berarti "buruk" — ada situasi di mana kamu memang hanya butuh library UI tanpa framework. Tapi untuk proyek fullstack modern, Next.js jauh lebih praktis.

🚀 Instalasi Next.js (JavaScript)

Sebelum mulai coding, kita harus install Next.js dulu. Penting: kita akan pakai JavaScript, bukan TypeScript. Perhatikan pilihan-pilihan di bawah ini.

Langkah 1 — Pastikan Node.js Terinstall

Cek apakah Node.js sudah terinstall di komputer kamu:

Terminal / Command Prompt
# Cek versi Node.js (harus v18 atau lebih baru)
node --version
# Output contoh: v20.11.0 ✅

# Cek npm juga
npm --version
# Output contoh: 10.2.4 ✅
⚠️ Belum punya Node.js? Download dari nodejs.org — pilih versi LTS (Long Term Support). Install seperti biasa, lalu restart terminal.

Langkah 2 — Buat Proyek Next.js Baru

Jalankan perintah ini di terminal:

Terminal — buat proyek baru
npx create-next-app@latest nama-proyek-kamu

Setelah dijalankan, akan muncul pertanyaan interaktif. Jawab seperti ini untuk menggunakan JavaScript (bukan TypeScript):

🎯 Pilihan yang BENAR untuk JavaScript
✔ Would you like to use TypeScript? … No    ← PILIH NO ❗
✔ Would you like to use ESLint? … Yes       ← Yes (bantu cek error)
✔ Would you like to use Tailwind CSS? … No ← No (kita pakai CSS biasa)
✔ Would you like your code inside a `src/` directory? … No  ← No (langsung di app/)
✔ Would you like to use App Router? (recommended) … Yes  ← YES ❗ (wajib!)
✔ Would you like to use Turbopack for next dev? … Yes   ← Yes (dev lebih cepat)
✔ Would you like to customize the import alias? … No   ← No (default saja)
💡 Kenapa pilih JavaScript, bukan TypeScript?
  • Lebih mudah dipahami — tidak ada tipe data tambahan yang membingungkan
  • Fokus ke konsep — kita belajar Next.js, bukan TypeScript
  • File extension .js — bukan .ts atau .tsx
  • TypeScript bisa dipelajari nanti setelah paham Next.js

Langkah 3 — Masuk ke Folder dan Jalankan

Terminal — jalankan proyek
# Masuk ke folder proyek
cd nama-proyek-kamu

# Jalankan development server
npm run dev

# Output:
#   ▲ Next.js 15.x.x
#   - Local:   http://localhost:3000    ← BUKA INI DI BROWSER!
#   ✓ Ready in 2.3s

Buka http://localhost:3000 di browser — kamu akan melihat halaman default Next.js! 🎉

Langkah 4 — Struktur Folder yang Dihasilkan

Setelah install, folder proyek kamu akan terlihat seperti ini:

Struktur folder proyek Next.js (JavaScript)
nama-proyek-kamu/
├── app/
│   ├── favicon.ico          ← ikon tab browser
│   ├── globals.css           ← CSS global
│   ├── layout.js             ← ✅ .js bukan .tsx!
│   ├── page.js               ← ✅ halaman utama
│   └── page.module.css       ← CSS khusus halaman
├── public/
│   ├── next.svg              ← gambar statis
│   └── vercel.svg
├── .eslintrc.json            ← konfigurasi ESLint
├── .gitignore
├── jsconfig.json             ← ✅ jsconfig (bukan tsconfig!)
├── next.config.mjs           ← konfigurasi Next.js
├── package.json              ← daftar dependencies
└── README.md
💡 Perhatikan! Semua file menggunakan extension .js (JavaScript), bukan .tsx (TypeScript). Ini karena kita pilih "No" untuk TypeScript saat instalasi. File konfigurasi juga jsconfig.json bukan tsconfig.json.

Langkah 5 — Edit Halaman Pertama

Buka app/page.js, hapus semua isinya, lalu ganti dengan kode sederhana ini:

app/page.js — Halaman pertama kamu!
// Ini adalah halaman utama — yang tampil di localhost:3000/

export default function Home() {
  return (
    <main>
      <h1>🎓 Halo Mahasiswa!</h1>
      <p>Ini adalah proyek Next.js pertama saya.</p>
      <p>Menggunakan <strong>JavaScript</strong>, bukan TypeScript.</p>
    </main>
  );
}

Simpan file → browser otomatis refresh → kamu akan melihat teks baru! 🎉

Perintah-Perintah Penting

Perintah yang sering dipakai
# 1. Jalankan development server (untuk ngoding)
npm run dev

# 2. Build untuk production (untuk deploy)
npm run build

# 3. Jalankan hasil build (untuk testing production)
npm start

# 4. Cek kualitas kode
npm run lint
🚫 Jangan Salah Pilih!

Kalau kamu tidak sengaja pilih TypeScript = Yes, file-file akan berekstensi .tsx dan kamu harus paham tipe data TypeScript. Kalau sudah terlanjur, hapus folder dan jalankan npx create-next-app@latest lagi dengan pilihan yang benar.

💻 Contoh Kode

Sekarang mari lihat perbedaannya secara langsung dalam kode. Perhatikan betapa lebih simpelnya Next.js!

1. Routing: React Biasa vs Next.js

Di React biasa, kamu harus install dan konfigurasi react-router-dom:

src/App.js — React biasa (Create React App)
// Harus install dulu: npm install react-router-dom
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import Home from './pages/Home';
import About from './pages/About';
import Contact from './pages/Contact';

// Setiap halaman baru = tambah import + route manual
function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
        <Route path="/contact" element={<Contact />} />
      </Routes>
    </BrowserRouter>
  );
}

Di Next.js, cukup buat file di folder app/ — routing otomatis!

app/page.js — Next.js (Halaman utama "/")
// Tidak perlu install react-router!
// File ini otomatis jadi halaman "/" (homepage)

export default function Home() {
  return <h1>Selamat Datang!</h1>;
}
app/about/page.js — Next.js (Halaman "/about")
// Buat folder "about" + file "page.js" = halaman "/about"
// Sesimpel itu! Tidak perlu konfigurasi router.

export default function About() {
  return <h1>Tentang Kami</h1>;
}

2. API Call: React Biasa vs Next.js

Di React biasa, kamu butuh server terpisah (misalnya Express.js) untuk API:

server.js — Express.js (server terpisah untuk React biasa)
// Harus buat project terpisah untuk backend!
const express = require('express');
const app = express();

// Endpoint untuk mengambil data users
app.get('/api/users', (req, res) => {
  res.json([
    { id: 1, nama: 'Budi' },
    { id: 2, nama: 'Siti' },
  ]);
});

app.listen(3001); // Port berbeda dari React!

Di Next.js, API langsung di dalam proyek yang sama:

app/api/users/route.js — Next.js (Built-in API)
// Tidak perlu Express.js atau server terpisah!
// File ini otomatis jadi endpoint "/api/users"

export async function GET() {
  const users = [
    { id: 1, nama: 'Budi' },
    { id: 2, nama: 'Siti' },
  ];

  return Response.json(users);
}

3. Hello World — Perbandingan Langsung

React biasa — src/App.js
import './App.css';

function App() {
  return (
    <div>
      <h1>Hello World</h1>
      <p>React biasa</p>
    </div>
  );
}

export default App;
Next.js — app/page.js
export default function Home() {
  return (
    <div>
      <h1>Hello World</h1>
      <p>Next.js</p>
    </div>
  );
}

// Sama saja! Bedanya
// hanya di struktur folder
💡 Perhatikan! Kode komponen di React biasa dan Next.js itu sama persis. Yang berbeda adalah bagaimana framework mengatur routing, build, dan fitur-fitur lainnya. Next.js tetap menggunakan React di dalamnya.

⚖️ Perbandingan Detail

❌ Tanpa Next.js (React Biasa)

Struktur folder & setup:

my-react-app/
├── public/
│   └── index.html
├── src/
│   ├── App.js
│   ├── index.js
│   ├── pages/
│   │   ├── Home.js
│   │   └── About.js
│   └── routes.js    ← manual!
├── webpack.config.js ← manual!
├── .babelrc           ← manual!
└── server.js          ← terpisah!
  • Install banyak package tambahan
  • Konfigurasi routing manual
  • Backend terpisah
  • Setup build rumit

✅ Dengan Next.js

Struktur folder & setup:

my-nextjs-app/
├── app/
│   ├── page.js       ← homepage
│   ├── about/
│   │   └── page.js   ← /about
│   ├── api/
│   │   └── users/
│   │       └── route.js ← API!
│   └── layout.js
├── public/
└── next.config.js
  • Minimal package — sudah lengkap
  • Routing otomatis dari folder
  • Backend built-in
  • Build 1 perintah

🤔 Kapan Pakai React Biasa vs Next.js?

Pakai React Biasa kalau:

  • Membuat widget kecil di dalam website lain
  • Proyek embedded (misal: Chrome extension)
  • Sudah punya backend sendiri dan hanya butuh UI
  • Belajar dasar-dasar React tanpa kompleksitas

Pakai Next.js kalau:

  • Membuat website/aplikasi fullstack
  • Butuh SEO yang bagus (blog, e-commerce)
  • Mau deploy cepat dan mudah
  • Proyek baru — ini jadi standar industri
  • Butuh API backend di proyek yang sama

⚠️ Kesalahan Umum

Hindari kesalahan-kesalahan ini saat belajar React dan Next.js:

✗ Kesalahan #1: "React dan Next.js itu beda total"

Banyak pemula berpikir React dan Next.js adalah dua hal yang berbeda. Padahal Next.js IS React — Next.js menggunakan React di dalamnya. Setiap komponen yang kamu tulis di Next.js adalah komponen React biasa.

✓ Benar: Next.js = React + fitur tambahan (routing, API, SSR)

✗ Kesalahan #2: Install react-router di Next.js

// ❌ JANGAN lakukan ini di Next.js!
npm install react-router-dom

// Next.js sudah punya routing sendiri.
// Cukup buat file di folder app/ dan routing otomatis jalan.

✓ Benar: Di Next.js, buat folder + page.js untuk routing. Tidak perlu library tambahan.

✗ Kesalahan #3: Pakai Create React App untuk proyek yang butuh SSR

Create React App (CRA) hanya menghasilkan aplikasi client-side rendered. Kalau kamu butuh SSR (Server-Side Rendering) untuk SEO atau performa, CRA bukan pilihan yang tepat.

✓ Benar: Gunakan npx create-next-app@latest untuk proyek yang butuh SSR.

✗ Kesalahan #4: Berpikir komponen di Next.js berbeda dari React

// Komponen ini bisa dipakai di React biasa DAN Next.js
// Tidak ada bedanya! Sama-sama komponen React.

function Tombol({ label, onClick }) {
  return (
    <button onClick={onClick}>
      {label}
    </button>
  );
}

✓ Benar: Komponen React = komponen Next.js. Skill React kamu tetap 100% berguna di Next.js.

✏️ Latihan

Coba kerjakan latihan berikut untuk memastikan kamu paham perbedaan React dan Next.js:

Latihan 1 — Halaman Utama

Buat file app/page.js di proyek Next.js yang menampilkan teks "Selamat Datang di Next.js!" dengan heading <h1> dan paragraf penjelasan singkat.

Petunjuk: app/page.js
export default function Home() {
  return (
    <main>
      <h1>Selamat Datang di Next.js!</h1>
      <p>Ini adalah halaman utama.</p>
    </main>
  );
}

Latihan 2 — Halaman About

Buat file app/about/page.js untuk membuat halaman /about. Tampilkan nama kamu, jurusan, dan deskripsi singkat. Perhatikan: kamu tidak perlu mengatur routing — Next.js otomatis membuatnya!

Petunjuk: app/about/page.js
export default function About() {
  return (
    <main>
      <h1>Tentang Saya</h1>
      <p>Nama: [Nama Kamu]</p>
      <p>Jurusan: Teknik Informatika</p>
    </main>
  );
}

Latihan 3 — Tantangan: Bandingkan Routing

Tulis di kertas atau text editor, bandingkan langkah-langkah membuat 3 halaman (Home, About, Contact) di:

  1. React biasa — berapa file yang perlu dibuat? Package apa yang harus di-install?
  2. Next.js — berapa file yang perlu dibuat? Apakah perlu install package tambahan?

Petunjuk: Di React biasa kamu butuh setidaknya 5-6 file (3 halaman + App.js + router config + index.js). Di Next.js? Cukup 3 file page.js di masing-masing folder.

📋 Ringkasan

Berikut poin-poin penting yang sudah kamu pelajari di halaman ini:

  • ✅ React adalah library untuk membuat UI, Next.js adalah framework fullstack
  • ✅ Next.js = React + routing otomatis + API backend + build system + SSR/SSG
  • ✅ Komponen Next.js tetap komponen React biasa — skill React kamu tetap berlaku
  • ✅ Routing di Next.js otomatis dari struktur folder (app/)
  • ✅ Untuk proyek baru, gunakan Next.js karena lebih lengkap dan sudah jadi standar industri
  • ✅ Jangan install react-router di Next.js — routing sudah built-in
  • ✅ Next.js mendukung API route tanpa perlu server terpisah
  • ✅ Deployment Next.js jauh lebih mudah (Vercel 1-click atau Docker)
